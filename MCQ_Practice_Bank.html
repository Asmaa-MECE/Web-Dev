32 questions · Selectors · Specificity · Flexbox · Grid · Media Queries

🎯 Selectors & Specificity
1 Which <p> element turns green?
<div class="box"> <p>Katmer</p> </div> <p>Kunafa</p> .box p { color: green; }
✅ Katmer — .box p is a descendant selector: it targets any <p> inside an element with class box. "Kunafa" is outside .box, so it is not selected.
2 Which elements become underlined?
<ul class="menu"> <li>Iskender</li> <li><span>Baklava</span></li> </ul> .menu > li { text-decoration: underline; }
✅ Both <li> elements (Iskender and the li wrapping Baklava) — > is the direct child combinator. Both <li>s are direct children of .menu. The <span> itself is not underlined by this rule (though it inherits the underline from the parent li).
3 Which elements are red?
<h1>Doha</h1> <h2>Istanbul</h2> <p>Hello</p> h1, p { color: red; }
✅ Doha (h1) and Hello (p) — the comma groups selectors. Istanbul (h2) is not in the selector list, so it is unaffected.
4 What happens when the mouse pointer is placed over the link?
<a class="link" href="#">Corniche Walk</a> .link { color: black; } .link:hover { color: orange; }
✅ The text color changes from black → orange when hovered. :hover is a pseudo-class that applies only when the mouse is over the element.
5 What is the final text color?
<p id="x" class="y">Qatar</p> p { color: blue; } .y { color: green; } #x { color: red; }
✅ Red — Specificity wins: ID (#x) = 100 points > Class (.y) = 10 points > Element (p) = 1 point. The ID rule has the highest specificity, so it wins regardless of order.
6 If the screen width is 500px, what is the font size?
<div class="card">Al Bayt Stadium</div> .card { font-size: 16px; } @media (max-width: 600px) { .card { font-size: 12px; } }
✅ 12px — 500px is less than 600px, so the media query condition max-width: 600px is true and the override applies.
7 Will A, B, and C appear in a row or stacked vertically?
<div class="row"> <div>A</div><div>B</div><div>C</div> </div> .row { display: flex; }
✅ In a row (side by side) — display: flex sets flex-direction: row by default, placing all child divs horizontally.
8 Which list items become purple?
<ul> <li>Tea</li> <li>Coffee</li> <li>Karak</li> <li>Turkish Coffee</li> </ul> li:nth-child(2n) { color: purple; }
✅ Coffee (2nd) and Turkish Coffee (4th) — nth-child(2n) selects every even child (n=1→2, n=2→4…). Same as nth-child(even).
9 What is the height at 900px screen width? What about 1100px?
.flag { height: 200px; } @media (min-width: 800px) { .flag { height: 400px; } } @media (min-width: 1000px) { .flag { height: 600px; } }
✅ 900px screen → 400px height (800px query fires, 1000px does not). 1100px screen → 600px height (both queries fire; the last one wins — cascade order).
10 How are the three items distributed horizontally?
<div class="bar"> <span>Souq</span><span>Corniche</span><span>Metro</span> </div> .bar { display:flex; justify-content: space-between; width:300px; border:1px solid; }
✅ Souq is flush left, Metro is flush right, Corniche is centered — space-between places the first item at the start, the last at the end, and distributes equal gaps between the remaining items.
11 Approximately how much width does A get compared to B?
.row { display:flex; width: 300px; } .a { flex-grow: 1; } .b { flex-grow: 2; }
✅ A gets ~100px (1/3), B gets ~200px (2/3) — flex-grow ratios: 1+2=3 total parts. A takes 1 part, B takes 2 parts of the available 300px.
12 How many columns are created, and what does 1fr mean?
.grid { display:grid; grid-template-columns: 100px 1fr; width:320px; }
✅ 2 columns — Column 1 is a fixed 100px. Column 2 is 1fr = 220px (320px − 100px). fr = fraction of the remaining space after fixed columns are subtracted.
13 Where does the .hero element appear?
.g { display:grid; grid-template-columns: 200px 1fr; } .hero { grid-column: 1 / -1; }
✅ It spans the full width — both columns — 1 / -1 means "from the first grid line to the last grid line." In a 2-column grid that is all columns.
14 At 600px screen width, what color is the text?
<p class="msg">Hello Doha</p> .msg { color: blue; } @media (max-width: 700px) { p { color: red; } }
✅ Blue — Both rules apply at 600px. But .msg (class selector, specificity 10) beats p (element selector, specificity 1). Higher specificity wins, so blue applies.
15 What is the total outer width of the card?
.card { width: 200px; padding: 20px; border: 10px solid; box-sizing: content-box; }
✅ 260px — With box-sizing: content-box (the default), width only sets the content area. Total = content (200) + padding left+right (20+20) + border left+right (10+10) = 260px.
16 How many columns does A occupy?
.grid { display:grid; grid-template-columns: 1fr 1fr 1fr; } .a { grid-column: span 2; }
✅ 2 columns — span 2 tells the item to occupy 2 column tracks from its starting position.
17 What is the font-size at 700px, 550px, and 480px?
.box { font-size: 16px; } @media (max-width: 900px) { .box { font-size: 14px; } } @media (max-width: 600px) { .box { font-size: 18px; } } @media (max-width: 500px) { .box { font-size: 12px; } }
✅ 700px → 14px (≤900 fires, ≤600 does not). 550px → 18px (≤900 and ≤600 both fire; last applied rule wins = 18px). 480px → 12px (all three fire; last = 12px).
18 What is the final color?
<p id="t" style="color: orange;">Karak time</p> #t { color: blue; } p { color: red; }
✅ Orange — Inline styles (specificity 1000) beat everything: ID (100), class (10), element (1). Inline style wins unconditionally.
19 Describe the page layout — who is top, left, right, bottom?
grid-template-areas: "h h" "a m" "f f";
✅ h = header (top, full width) · a = aside (left) · m = main (right) · f = footer (bottom, full width). Each letter in quotes = a named grid area cell. Repeating the same letter spans multiple columns.
20 How will the text "Spices" and "Oud" appear?
.souq p { font-weight: bold; } <div class="souq"> <p>Spices</p> <p>Oud</p> </div>
✅ Both Spices and Oud will be bold — .souq p targets all <p> descendants of .souq. Both paragraphs are inside .souq, so both get font-weight: bold.
21 How will the two dhow boat images be arranged?
.dhow-container { display: flex; } <div class="dhow-container"> <img src="dhow.jpg" alt="A dhow boat"> <img src="dhow2.jpg" alt="Another dhow boat"> </div>
✅ Side by side (horizontally) — display: flex with default flex-direction: row places all children in a horizontal row.
22 What will be the layout of the room?
.room { display: grid; grid-template-columns: 100px 100px; } <div class="room"> <div>Guest 1</div> <div>Guest 2</div> </div>
✅ Guest 1 and Guest 2 side by side in 2 equal 100px columns — Grid fills columns left to right. Two columns defined = two items placed in the same row.
23 How will "Walk" and "Cycle" be displayed?
.corniche { display: flex; flex-direction: column; } <div class="corniche"> <span>Walk</span> <span>Cycle</span> </div>
✅ Stacked vertically — flex-direction: column makes the main axis vertical, so items flow top-to-bottom: Walk on top, Cycle below.
24 Which paragraph's text will be gray?
.pearl-monument > p { color: gray; } <div class="pearl-monument"> <p>A symbol of Qatar.</p> </div> <p>Located in Doha.</p>
✅ "A symbol of Qatar." only — > is the direct child combinator. The second <p> is outside .pearl-monument and is not selected.
25 What is the width ratio between the entrance and the gallery?
.museum-layout { display: grid; grid-template-columns: 1fr 3fr; } <div class="museum-layout"> <div class="entrance"></div> <div class="gallery"></div> </div>
✅ Entrance : Gallery = 1 : 3 — Total = 4 fr parts. Entrance takes 1/4 of the width, Gallery takes 3/4. e.g., 800px → entrance = 200px, gallery = 600px.
26 Which list item will be in italics?
li:nth-child(2) { font-style: italic; } <ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol>
✅ "Second step" — nth-child(2) selects the second child element.
27 How will the station divs be aligned vertically within the metro-line?
.metro-line { display: flex; align-items: center; } .station { height: 50px; } .station:nth-child(2) { height: 100px; } <div class="metro-line"> <div class="station"></div> <div class="station"></div> <div class="station"></div> </div>
✅ All stations are vertically centered — align-items: center aligns items along the cross axis (vertical in a row flex). The tall station (100px) sets the row height; the shorter ones (50px) are centered within it.
28 If the container has extra space, how will that space be distributed among the items?
.traditional-pattern { display: flex; } .item { flex-grow: 1; } .item:nth-child(3) { flex-grow: 3; } <div class="traditional-pattern"> <div class="item">1</div> <div class="item">2</div> <div class="item">3</div> <div class="item">4</div> </div>
✅ Total grow units = 1+1+3+1 = 6. Items 1, 2, 4 each get 1/6 of extra space. Item 3 gets 3/6 (half) of the extra space. Item 3 is significantly wider than the others.
29 In what visual order will the articles appear?
.news-feed { display: flex; } .featured { order: -1; } <div class="news-feed"> <article>Story 1</article> <article class="featured">Story 2 (Featured)</article> <article>Story 3</article> </div>
✅ Visual order: Story 2 (Featured) → Story 1 → Story 3. Default order is 0. order: -1 moves .featured before all items with order 0 or higher. HTML source order is unchanged.
30 What determines the width and height of the child div?
.parent { width: 200px; height: 200px; background-color: lightblue; } .child { width: 50%; height: 10vw; background-color: maroon; } <div class="parent"><div class="child"></div></div>
✅ Width = 100px (50% of parent 200px). Height = 10vw (10% of viewport width — e.g., on a 1000px screen = 100px). Note: % width is relative to parent; vw is relative to the viewport, not the parent.
31 How will the images in the gallery behave when you resize the browser window?
.gallery { display: grid; grid-template-columns: repeat(auto-fit, minmax(150px, 1fr)); gap: 10px; } <div class="gallery"> <img src="1.jpg"> <img src="2.jpg"> <img src="3.jpg"> <img src="4.jpg"> <img src="5.jpg"> <img src="6.jpg"> </div>
✅ Responsive auto-fitting grid — auto-fit creates as many columns as fit. Each column is at least 150px wide and stretches equally with 1fr. On wide screens: 6 columns. As the window narrows, columns drop to 5, 4, 3, 2, 1 — no media queries needed.
32 When you hover over the word "Click", what color will it be?
a:hover { color: red; } span { color: blue; } <a><span>Click</span></a>
✅ Blue — span { color: blue } directly targets the <span> (specificity: element = 1). a:hover only sets color on the <a>, not on the <span> inside. The <span>'s direct rule wins over inherited color from the parent <a>.

1 Which selector sets the page background color?
✅ body { background: #f6f6f6; } — The body selector applies a light gray background to the entire page.
2 What does max-width: 900px do for .container?
✅ It caps the container width at 900px. On screens wider than 900px the container stays 900px and is centered with margin: 0 auto. On narrower screens the container shrinks to fit.
3 Which CSS rule makes the header items go side-by-side?
✅ header { display: flex; } — Flexbox places children (.logo and .tag) in a horizontal row by default.
4 What property creates space between items in the header?
✅ gap: 12px — The gap property on a flex container adds space between flex items (without affecting outer edges).
5 Which selector styles all three cards?
✅ .card { ... } — The class selector targets every element with class="card". All three menu item divs share this class.
6 What changes visually when you hover over .card?
✅ Two things happen: (1) The card lifts slightly — transform: translateY(-2px) moves it 2px upward. (2) The border darkens — border-color: #999 (was #ddd). The transition makes both changes animate smoothly over 150ms.
7 Which selector makes the "Order" link look like a button?
✅ .btn { ... } — The class selector applied to the <a> tags gives them a dark background, white text, padding, and border-radius — making them look like buttons.
8 What does text-decoration: none; do in .btn?
✅ It removes the default underline from the <a> link — making it look like a button rather than a text hyperlink.
9 In .grid, what does grid-template-columns: repeat(3, 1fr) mean?
✅ Creates 3 equal-width columns. repeat(3, 1fr) is shorthand for 1fr 1fr 1fr — each column gets 1/3 of available space.
10 How many cards appear per row on a wide screen?
✅ 3 cards per row — The grid has repeat(3, 1fr) = 3 columns. All 3 menu cards fill exactly one row on desktop.
11 What happens to the layout when the screen width is 650px?
✅ The @media (max-width: 700px) breakpoint fires. Two things change: (1) The grid collapses to 1 column — cards stack vertically. (2) The header switches to flex-direction: column — logo and tag stack vertically.
12 Which media query rule changes the header layout on small screens?
✅ @media (max-width: 700px) { header { flex-direction: column; align-items: flex-start; } } — This switches the header from horizontal to vertical layout on narrow screens.
13 If you remove border-radius: 10px from .card, what changes?
✅ The cards get sharp square corners instead of rounded ones. No layout change — only the visual shape of the card borders changes.
14 What is the effect of box-sizing: border-box; on *?
✅ It makes width and height include padding and border for every element. With content-box (default), padding/border are added on top of the declared width, causing overflow. border-box makes sizing predictable and is considered best practice.
15 If you add a { color: green; } at the bottom — will the button text stay white or turn green? Why?
✅ Turn green — a { color: green } (element, specificity 1) vs .btn { color: white } (class, specificity 10). But if a { color: green } is added later and specificity is equal or lower, class wins. However since .btn already sets color: white with higher specificity (class > element), the button stays white. The class rule wins.
16 If you add .card .price { color: red; }, which elements change?
✅ All three price divs (12 QAR, 15 QAR, 18 QAR) turn red — .card .price is a descendant selector targeting any element with class price inside any element with class card.
17 If you add .grid { gap: 40px; }, what happens to spacing?
✅ The gaps between cards increase from 12px → 40px. Cards remain the same size but are spaced much further apart. Total grid width stays the same — cards shrink slightly to accommodate larger gutters.
18 If you change the media query from (max-width: 700px) to (max-width: 500px), what changes for tablets?
✅ Tablets (~600-700px wide) will now show the 3-column desktop layout instead of the single-column mobile layout. The breakpoint only triggers on screens ≤500px, which excludes most tablets.
19 If you add a 4th .card, how does the grid behave on desktop?
✅ The first 3 cards fill row 1; the 4th card starts a new row and occupies the first column. CSS Grid automatically creates new rows for overflow items using the implicit grid.
20 If you add header { justify-content: center; }, how does the header change?
✅ The logo and tag move to the center of the header. Currently they are spread apart with space-between; adding center overrides this, grouping them together in the middle.
21 If you remove display: grid; from .grid, what happens to the cards?
✅ The cards stack vertically in a single column. Without display: grid, the div children use their default display: block behavior — each card takes the full width and sits below the previous one.
