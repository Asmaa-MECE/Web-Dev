HTML Fundamentals

Page Structure · Semantic Tags · Tables · Media · Forms & Input Types

1

Introduction to HTML

What is HTML?

  • HyperText Markup Language — a text file containing markup tags
  • Describes the content & logical structure of a web page
  • HTML = Content/Structure | CSS = Style/Presentation | JS = Behavior
  • An HTML page can reference CSS, JavaScript, images, audio, and video

HTML Page Structure

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" href="style.css">
    <script src="app.js"></script>
  </head>
  <body>
    <h1>Heading</h1>
    <p>Paragraph</p>
  </body>
</html>

What HTML Can Do

  • Define logical document structure
  • Collect user input via Forms
  • Display tabular data using Tables
  • Embed images, audio, video
  • Draw graphics using <canvas>
2

Semantic Page Structure

Semantic Tags

Use semantic tags to define the logical structure of the page in a meaningful way. Use CSS to arrange elements into the desired layout.

Semantic Tags Reference

TagPurpose
<header>Introductory content, logo, nav links — typically at the top
<nav>Primary navigation links (often inside header)
<main>The main content of the page (unique per page)
<section>Thematic grouping of content, like newspaper sections
<article>Self-contained content that makes sense on its own (e.g., blog post)
<aside>Sidebar content — can be removed without reducing main meaning
<footer>Copyright, contact info, social links
<div>Generic container — use when no semantic tag applies

Article vs Section

<article> = self-contained post (makes sense when read alone). <section> = thematic grouping. Think newspaper: many articles inside each section. An article can contain sections (intro, body, summary).

HTML Tables

Table Structure

Tables represent tabular data and are built with semantic sections.

<table>
  <thead>
    <tr><th>Col 1</th><th>Col 2</th></tr>
  </thead>
  <tfoot>  <!-- comes BEFORE tbody in code -->
    <tr><td>Footer 1</td><td>Footer 2</td></tr>
  </tfoot>
  <tbody>
    <tr><td>Cell 1.1</td><td>Cell 1.2</td></tr>
  </tbody>
</table>

⚠️ Important: tfoot placement

Even though <tfoot> appears before <tbody> in the HTML code, the browser renders the footer at the bottom of the table.

CSS Introduction

What is CSS?

  • Cascading Style Sheets — controls presentation and layout of HTML
  • Separates content (HTML) from design (CSS)
  • Flexible, cross-platform, standards-based (W3C)

Three Ways to Include CSS

MethodHowBest For
Inline<p style="color:red">Quick one-off overrides
Embedded<style> in <head>Single-page styles
External<link rel="stylesheet" href="style.css">✅ Recommended — reusable, cached

The Cascade

Last rule applied wins. Styles are applied in order found — later rules overwrite earlier ones in case of conflict. Inline styles win over all others.

Inheritance

Child elements inherit properties from their parent. A style on body (e.g., font) applies to all descendants unless overridden by a more specific rule.

Box Model, Margin, Border & Padding

The Box Model

Every HTML element is a box with 4 layers from inside out: Content → Padding → Border → Margin.

Shorthand Margin & Padding Rules

/* 1 value: all 4 sides */
margin: 0.8rem;

/* 2 values: top&bottom | left&right */
margin: 10px 20px;

/* 4 values: top right bottom left (clockwise) */
margin: 1px 3px 5px 7px;

/* Individual sides */
margin-top: 10px;  padding-left: 1rem;

Borders

/* shorthand: width | style | color */
border: 2px solid #3ecf8e;
border-radius: 8px;   /* rounds corners */
border-top: 1px dashed #ccc; /* individual side */

Flexbox

What is Flexbox?

A one-dimensional layout model for rows OR columns. Flex container stretches/shrinks items to fill space.

.flex-container {
  display: flex;
  gap: 1rem;
  justify-content: center;
}

Flex Container Properties

PropertyValues / Effect
flex-directionrow (default) | column — direction of items
flex-wrapnowrap (default) | wrap — allow multiple lines
justify-contentAligns items along main axis (start, center, space-between, space-around)
align-itemsAligns items along cross axis (stretch default, center, flex-start)
align-contentDistributes space between wrapped lines
gapSpace between flex items

Flex Item Properties

PropertyEffect
flex-growHow much item grows to fill space (0 = don't grow)
flex-shrinkHow much item shrinks when space is limited
flex-basisDefault size before space is distributed
flex: grow shrink basisShorthand — e.g., flex: 1 1 200px
orderChanges display order of item
align-selfOverrides align-items for a single item
F

CSS Grid

What is CSS Grid?

A two-dimensional layout system for both rows AND columns. Best for macro/page-level layouts.

.container {
  display: grid;
  grid-template-columns: 2fr 1fr;
  grid-template-rows: auto 1fr auto;
  gap: 1rem;
}

Grid Properties

PropertyEffect
grid-template-columnsDefine column sizes: px, %, fr, auto
grid-template-rowsDefine row sizes
grid-gap / gapSpace between grid tracks (gutters)
grid-column: 2/4Item spans from line 2 to line 4
grid-column: 1/-1Spans all available columns
grid-area: headerName the item for use in template-areas
grid-template-areasNamed placement — best for readable layouts

Named Grid Areas — Best Practice

.container {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main  aside"
    "footer footer footer";
}
header { grid-area: header; }
main   { grid-area: main; }

Grid vs Flexbox

Grid = 2D (rows + columns) → use for page-level macro layouts. Flexbox = 1D (row OR column) → use for component-level micro layouts like navbars and cards. In practice, combine both: Grid for page layout, Flexbox inside grid cells.

Media Queries

Responsive Web Design (RWD)

Use media queries to serve different layouts for different screen sizes. Use a mobile-first approach — start with single-column, then add columns as screen widens.

/* Apply 2-column grid only on screens > 700px */
@media screen and (min-width: 700px) {
  .page {
    display: grid;
    grid-template-columns: 2fr 1fr;
  }
}

Introduction to JavaScript

What is JavaScript?

  • Platform-independent scripting language — runs anywhere
  • Lightweight but powerful interpreted language
  • Supports both functional and object-oriented programming
  • Current version: ES 2023 (ECMAScript 2023)
  • Syntactically a C-family language (like Java / C#)

Where it runs

  • Client-side – embedded in HTML, run by the browser
  • Server-side – using Node.js
  • Desktop apps – e.g. Electron.js
  • Mobile apps – e.g. React Native

What JS Can Do

  • Handle client-side events (clicks, mouse-over)
  • Manipulate the DOM (read/add/edit/delete HTML elements)
  • Validate form input before server submission
  • Perform computations, sorting, animation
  • Async server calls (AJAX) without page reload

Full Stack Note

In a modern stack (MEAN/React/Next.js): JavaScript is the common language throughout the full stack, and JSON is the common data format.

2

Data Types in JavaScript

Declaring Variables

  • Use const by default — creates a block-scoped, read-only variable
  • Use let when the value needs to change (mutable variable)
  • Variable names are case-sensitive
  • JS is loosely typed — the type comes from the assigned value
const height = 110;      // number
let count = 5;
count = 6;            // OK — let is mutable

The 7 Primitive Types

TypeExampleNotes
number5, 3.14Integer & float same type
bigint9007199254740991nVery large integers

booleantrue, false–
undefinedlet x;Declared but not assigned
nulllet x = null;Intentional "no value"; typeof → object
functionfunction f(){}–

Everything else is an object. Use typeof x to check type at runtime.

undefined vs. null

  • undefined – declared but never assigned
  • null – explicitly assigned to mean "no value"
  • typeof undefined → "undefined"
  • typeof null → "object" (historical JS quirk)

NaN (Not a Number)

  • Result of illegal/erroneous operations (e.g., 'A' * 2)
  • Toxic: any arithmetic with NaN → NaN
  • NaN === NaN is false
  • Use isNaN(value) to check

Operators in JavaScript

== vs === (IMPORTANT)

  • == → Loose equality: performs type conversion before comparing
  • === → Strict equality: NO type conversion — types must match too
0 == ""    // true  (type coercion)
0 === ""   // false (different types)

⚠️ Prefer === in practice

Web API

HTTP · REST · CRUD · Next.js Routes · Request & Response

1

What is a Web API?

Definition

  • Web API = Web-accessible Application Programming Interface (also known as REST Services)
  • A web service that accepts HTTP requests and returns structured data (JSON) instead of HTML
  • Programmatically accessible at a particular URL
  • Goal: interoperability between heterogeneous systems
  • Can be accessed by any client: browsers, mobile apps, IoT devices, other servers

Full Stack Architecture

Front-end (UI Components) ↔ Web API (JSON) ↔ Back-end App Logic ↔ Database. The Web API is the interface layer between client and server.

SPA Architecture

  • A Single Page Application (SPA) fits on a single web page
  • Page resources are retrieved at load time or loaded in response to user actions
  • The page does not reload at any point — updates are handled by JS DOM manipulation
2

HTTP & URLs

What is HTTP?

  • HyperText Transfer Protocol — the message protocol of the Web
  • Uses a Request/Response interaction model
  • Client sends a request → Server sends a response
  • HTTP is an application protocol, not just a transport protocol

URL Structure

// https://www.example.com:443/api/books?sort=title
// └─protocol─┘ └──host──┘ └port┘ └─path─┘ └─query─┘

Protocol  : http, https
Host      : domain name or IP address
Port      : e.g. 80 (http), 443 (https), 3000 (dev)
Path      : /api/books — identifies the resource
Query     : ?key=value — optional parameters after ?

HTTP Request Message

  • Request line — method, resource URI, protocol version
  • Headers — additional parameters (Content-Type, Authorization…)
  • Body — optional data (e.g., form data, JSON payload)

HTTP Response Message

  • Status line — protocol version, status code, phrase
  • Headers — metadata (Content-Type, etc.)
  • Body — the requested resource (HTML, JSON, image…)
3

REST Principles

REST = Representational State Transfer

  • Nouns (Resources) — identified by unique URIs: http://example.com/employees/123
  • Verbs (HTTP Methods) — GET, POST, PUT, PATCH, DELETE
  • Representations — data format returned (usually JSON, sometimes XML)

Naming Resources — Best Practices

// Use plural nouns for collections
/api/books               // collection
/api/books/ISBN-0011     // specific resource
/api/books/ISBN-0011/authors  // sub-collection

/api/classes             // collection
/api/classes/cmps350     // specific class
/api/classes/cmps350/students // sub-collection

Traversing the path from generic → specific = navigating the data hierarchy. Always use a dedicated /api prefix.

4

POST vs PUT

POST = server decides the new resource URI (not repeatable). PUT = client decides the resource URI (repeatable/idempotent). PUT is typically used for full replacement/update.

  • Run dev server: npm run dev

API Routes — route.js

  • Add a route.js file inside app/api/
  • Export async functions named by HTTP verb: GET, POST, PUT, DELETE
  • Each subfolder with route.js = one API endpoint

Route Handler Pattern

// app/api/books/route.js
export async function GET(request) {
  return new Response(
    JSON.stringify(books),
    { headers: { 'Content-Type': 'application/json' } }
  );
}

export async function POST(request) {
  const body = await request.json();
  books.push(body);
  return new Response(JSON.stringify(body), {status: 201});
}

Dynamic Routes & Query Parameters

Dynamic Route Segments — [folderName]

// File: app/api/books/[id]/route.js
// URL: GET /api/books/42 → params.id = "42"

export async function GET(request, { params }) {
  const book = books.find(b => b.id === Number(params.id));
  if (!book) return new Response('Not found', {status: 404});
  return new Response(JSON.stringify(book));
}

export async function DELETE(request, { params }) {
  books = books.filter(b => b.id !== Number(params.id));
  return new Response(null, {status: 204});
}

Catch-all Routes — [...slug]

  • [...slug] matches any number of path segments
  • app/blogs/[...filterBy] matches: /blogs/2023, /blogs/2023/3/10
  • Matched segments are available as an array in params

Query Parameters

// URL: /products?sortBy=price
export async function GET(request) {
  const { searchParams } =
    new URL(request.url);
  const sortBy = searchParams
    .get('sortBy'); // "price"
}

Introduction to React

What is React?

  • Open-source JavaScript library for building component-based User Interfaces (UI)
  • UI is composed of small, reusable components
  • A component encapsulates UI elements and the behavior associated with them
  • React components can be rendered on server-side or client-side
  • Open-sourced by Facebook mid-2013 — reactjs.org
  • Competing with Angular and Vue.js
  • Used by: Facebook, Instagram, Netflix, Dropbox, Outlook, Yahoo, Khan Academy…

Web App Architecture

  • A web app has pages
  • A page is composed of components (1 page → many components)
  • A component can be a tree of components
  • App → Pages → Components

React as a Declarative Model

  • UI is built using JavaScript functions
  • Each function defines a piece of the app's UI programmatically
  • As state changes, the UI automatically updates (Reactive UI)
  • Declarative — you define structure & states; React handles the DOM

React Component Libraries

  • Material-UI — React components with Material Design (mui.com)
  • Grommet — accessible component library (v2.grommet.io)
  • Blueprint — React-based UI toolkit (blueprintjs.com)

React Components & JSX

What is a React Component?

  • A component returns HTML elements to provide the UI
  • Encapsulates state (internal data) and functions to handle events
  • Component = UI + display logic
  • Components allow creating new custom "HTML tags"
  • Component name must start with a capital letter

What is JSX?

  • JSX = JavaScript XML — HTML-like markup to describe the component's UI
  • JSX allows writing HTML-like syntax which gets transformed to JavaScript objects
  • You can embed JavaScript expressions in JSX using {}

Creating & Using a Component

// Create a Welcome component
function Welcome(props) {
    return (<h1>Welcome to {props.appName}</h1>);
}

// Use the Welcome component

Props are read-only — the component can read them but cannot modify them.

3

Props

What are Props?

  • Props = data passed from the parent component to the child component
  • Props parameters are read-only — child cannot modify them
  • Work just like HTML attributes — they configure the component with different content
  • Makes components reusable with different data

Props Destructuring

Instead of accessing props.firstName you can destructure props into variables directly in the function signature:

// Without destructuring
function UserInfo(props) {
    return (
        <div>
            {props.firstName}
            {props.lastName}
        </div>
    );
}

// With destructuring (preferred)
function UserInfo({ firstName, lastName }) {
    return (
        <div>
            {firstName}
            {lastName}
        </div>
    );
}

Special "children" Prop

  • The children property holds the content between the component's opening and closing tags
  • A special property auto-added by React
// Usage
<Welcome name="Ali Faleh">
    <h2>Welcome to QU</h2>
    <img src="logo.png" />
</Welcome>

// Component definition
function Welcome({name, children}) {
    return (
        <>
            <h1>Welcome {name}</h1>
            {children}
        </>
    );
}

⚠️ Keys are Very Important in Lists

  • A key is a unique identifier to identify which list items have changed, been added, or deleted
  • Helps React determine which components need to be re-rendered (instead of re-rendering all components)
  • Increases performance — only updated components are re-rendered
  • Keys must be unique among siblings
5

State & useState Hook

What is State?

  • State = any value that can change over time
  • A component can store its own local data (state)
  • State is private and fully controlled by the component
  • Can be passed as props to children
  • UI = f(state) — UI is a visual representation of state
  • State changes trigger automatic update of the UI

useState Hook

  • Use useState hook to create a state variable and an update function
  • Never change state directly — always use the setter function
const [count, setCount] = useState(0);
// count    = state variable (initialized to 0)
// setCount = function to update it

// Calling setCount causes React to re-render
setCount(count + 1);

⚠️ Never assign directly: count = 5 — React will NOT re-render the UI!

State Variables as Change Notifiers

  • State variables must be declared using useState to act as Change Notifiers
  • They are observed by the React runtime
  • Any change triggers re-rendering of any function that reads the state variable
  • Both props and state changes trigger a render update

Observer Pattern: Like a celebrity with fans on Twitter. Fans (React runtime) observe the celebrity (state variable) and auto-update when something changes.

Imperative UI vs. Declarative UI (React)

Imperative (Traditional DOM)Declarative (React)
Manipulate DOM directly to change UI state Update state variables — React updates DOM automatically 
document.querySelector('#bulbImage')
  .src = 'bulb-on.png';
 
<input type="button"
  value={isBulbOn ? "Turn off" : "Turn on"}
  onClick={() => setIsBulbOn(!isBulbOn)} />

In React, UI is immutable — do NOT access/update UI elements directly. Instead, update state variables.

Virtual DOM

  • Virtual DOM = Pure JavaScript lightweight DOM, totally separate from the browser's slow DOM API
  • Every time a component updates its state or receives new props → a new virtual DOM tree is generated
  • New tree is diffed against the old tree
  • Produces a minimum set of changes to perform on the real DOM to bring it up to date
  • Result: excellent performance — only necessary DOM updates happen

Events & Forms

Handling Events

  • Event handling works similarly to how events are handled on DOM elements
  • Common events: onClick · onSubmit · onChange
  • Event handlers update state → React updates the UI
  • Every place a state variable is displayed is guaranteed to be auto-updated

Counter Component — State + Events

import React, { useState } from "react";

function Counter(props) {
    const [count, setCount] = useState(props.startValue);
    const increment = () => { setCount(count + 1); };
    const decrement = () => { setCount(count - 1); };

    return <div>
            Count: {count}
            <button type="button" onClick={increment}>+</button>
            <button type="button" onClick={decrement}>-</button>
        </div>
}
export default Counter;

// Use: <Counter startValue={3}/>

Forms with React — Controlled Components

const [state, setState] = useState({ email: "", password: "" });

const handleChange = e => {
    const name = e.target.name;
    const value = e.target.value;
    // Merge the object before change with updated property
    setState({ ...state, [name]: value });
};

const handleSubmit = e => {
    e.preventDefault();
    alert(JSON.stringify(state));
};

// Form UI
<form onSubmit={handleSubmit}>
    <input name="email" type="email" required
           value={state.email} onChange={handleChange} />
    <input name="password" type="password" required
           value={state.password} onChange={handleChange} />
    <input type="submit" />
</form>

Components Communication

Parent-Child Communication

  • Components are meant to be used together — most commonly in parent-child relationships
  • Parent → Child: Parent passes data down to the child via props
  • Child → Parent: Child notifies its parent of a state change via callbacks (parent must pass a callback function as a prop)

Callback Example — Child Notifying Parent

// Parent: passes onChange callback to child
function Main() {
  return <Counter
    startValue={3}
    onChange={count =>
      console.log(`Count from child: ${count}`)
    }
  />
}

// Child: calls props.onChange to notify parent
function Counter(props) {
    const [count, setCount] = useState(props.startValue);

    const increment = () => {
        const updatedCount = count + 1;
        setCount(updatedCount);
        props.onChange(updatedCount); // notify parent
    };

    return <div>
        Count: {count}
        <button type="button" onClick={increment}>+</button>
    </div>
}

Common Hooks

What are Hooks?

A Hook is a special function that lets you hook into React features such as state and lifecycle methods.

HookPurpose
useStateManage state inside a component
useEffectPerform side effects & hook into component lifecycle
useRefAccess DOM elements directly or persist mutable values

useEffect Hook

  • For doing stuff when a component mounts / unmounts / updates
  • Ideal for fetching data when the component is mounted
  • Takes two arguments: a callback function and an optional dependency array
2nd ArgumentBehavior
Not provideduseEffect runs on every re-render
[] (empty array)Runs once when component first mounts
[query] (with deps)Runs whenever query changes

useEffect Lifecycle

  • A) After initial rendering: useEffect invokes the callback (side-effect). Cleanup is NOT invoked.
  • B) On later renderings: Before next side-effect, useEffect invokes the cleanup from the previous run, then runs the current side-effect.
  • C) After unmounting: useEffect invokes the cleanup function one final time.
// Fetch data once when component mounts
useEffect(() => {
    async function fetchData() {
        const res = await fetch("https://api.github.com/users");
        setUsers(await res.json());
    }
    fetchData();
}, []); // empty array = run once on mount

// Fetch when 'query' state variable changes
useEffect(() => {
    async function fetchData() {
        const url = `https://hn.algolia.com/api/v1/search?query=${query}`;
        const res  = await fetch(url);
        const data = await res.json();
        setNews(data.hits);
    }
    fetchData();
}, [query]); // re-run whenever query changes

useRef Hook

  • Creates persisted mutable values and accesses DOM elements directly (e.g., focusing an input)
  • The value of the reference is persisted between re-renderings
  • Updating a reference does NOT trigger a re-render
  • useRef(initialValue) returns a reference object with a current property
// useRef for mutable values
const reference = useRef(0);
reference.current; // access value
reference.current = 5; // update value
// ↑ does NOT trigger re-render!
// useRef for DOM element access
const inputRef = useRef();

// Assign to element
<input ref={inputRef} />

// After mount: focus input
inputRef.current.focus();

useStateuseRef
Triggers re-render?✅ Yes — on every state change❌ No — value change is silent
Persists across renders?✅ Yes✅ Yes
Best useUI data that should update the screenDOM access, timers, non-visual data
